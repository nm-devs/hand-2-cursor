ISSUE #1: BIG BLACK RECTANGLE IN MIDDLE OF SCREEN
================================================================================

Location: utils/gesture_display.py, Line 32-35

CURRENT (BROKEN) CODE:
──────────────────────
def draw_sentence_display(frame, sentence, position= (10, 400)):
    # draw current sentence at bottom of frame
    x, y = position
    h, w, _ = frame.shape
    # draw background rectangle
    cv2.rectangle(frame, (x-5, y-30), (w-10, y+10), (0,0,0), -1)  <-- ALWAYS DRAWS!
    # label
    cv2.putText(frame, sentence, (x, y+30), cv2.FONT_HERSHEY_SIMPLEX, 1, (255,255,255), 2)
    return frame


PROBLEM:
────────
The function draws a black rectangle EVERY TIME, even when:
  - sentence is None
  - sentence is empty string ""
  - No gesture has been made yet

This creates a large black rectangle covering the middle/bottom of the screen
that never goes away.


FIX (SOLUTION):
───────────────
Add a check to only draw if sentence has content:

CHANGE FROM:
    def draw_sentence_display(frame, sentence, position= (10, 400)):
        # draw current sentence at bottom of frame
        x, y = position
        h, w, _ = frame.shape
        # draw background rectangle
        cv2.rectangle(frame, (x-5, y-30), (w-10, y+10), (0,0,0), -1)
        # label
        cv2.putText(frame, sentence, (x, y+30), cv2.FONT_HERSHEY_SIMPLEX, 1, (255,255,255), 2)
        return frame

CHANGE TO:
    def draw_sentence_display(frame, sentence, position= (10, 400)):
        # draw current sentence at bottom of frame
        if not sentence:  # <-- ADD THIS CHECK
            return frame
        
        x, y = position
        h, w, _ = frame.shape
        # draw background rectangle
        cv2.rectangle(frame, (x-5, y-30), (w-10, y+10), (0,0,0), -1)
        # label
        cv2.putText(frame, sentence, (x, y+30), cv2.FONT_HERSHEY_SIMPLEX, 1, (255,255,255), 2)
        return frame

ISSUE #2: GESTURES NOT BEING DETECTED (Hard to trigger)
================================================================================

Location: main.py, _process_prediction() method, Lines ~115-120

CURRENT (TOO STRICT) LOGIC:
────────────────────────────
    if confidence > 0.70:  # Only track confident predictions
        self.prediction_history.append(label)
    
    unique_predictions = len(set(self.prediction_history))
    
    if unique_predictions >= 3:  # 3+ different signs = likely gesture


PROBLEM:
────────
The threshold of "3+ different predictions" is TOO HIGH. This requires:
  1. Making 5-10 different hand shapes
  2. Each one > 70% confident
  3. All within the deque window (5 items max)

Result: User has to "jiggle" their hand A LOT before gestures work.

Real problem: The prediction_history is only populated if confidence > 0.70,
but if user goes straight to a gesture (open palm) without first signing,
the classifier gives low confidence, so history stays empty.


FIX (SOLUTION A - Recommend):
──────────────────────────────
LOWER the threshold from 3 to 2 unique predictions:

CHANGE FROM:
    if unique_predictions >= 3:  # 3+ different signs = likely gesture

CHANGE TO:
    if unique_predictions >= 2:  # 2+ different signs = likely gesture


This now requires only 2-3 quick hand shape changes to trigger gesture mode.
Much more user-friendly!


ALTERNATIVE FIX (SOLUTION B - More lenient):
──────────────────────────────────────────────
Allow gestures even with just 1 different prediction, but only after briefdelay:

    if unique_predictions >= 1:
        # Give user 0.5 seconds to complete the gesture
        if time.time() - self.first_motion_time > 0.2:
            gesture = self.gesture_detector.detect_gesture(hands_data)
            ...

This is more sensitive but might trigger accidentally. Solution A is safer.

================================================================================
TECHNICAL DETAILS (Why these fixes work)
================================================================================

WHY Black Rectangle Issue:

cv2.rectangle() draws IMMEDIATELY to the frame.
There was no guard checking if there's text to display.
So EVERY iteration, function draws black box.
Even if sentence is None or "", it still draws! Always!

Fix adds early return: if not sentence: return frame

This prevents drawing anything if there's no content.
Like a guard clause at the door.


WHY Gesture Detection Too Strict:

Original logic: Wait for 3+ different classifications
  - Assumes user is "exploring" multiple hand shapes
  - Actually happens when user is trying to sign different letters
  - By the time 3 different predictions occur, hand has moved too much
  - Gesture might have already passed!

New logic: Wait for 2 different classifications  
  - Much quicker to trigger (after 0.1-0.3 seconds of movement)
  - User shows hand shape A, then quickly shows gesture B
  - detection works!
